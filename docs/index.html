SWRL rules

  • Rule 1 — Team membership implies having a role
  • Rule 2 — Team, Goal, and Capability determine eligibility
  • Rule 3 — Eligibility implies assignment
  • Rule 4 — Assignment → Performance

Rule 1 — Team membership implies having a roleback to ToC or SWRL ToC

If an HI team exists and has a member agent, then the agent must be classified by some Role (hi:hasRole).

HI Team c(?) , has Member op(?,?) -> has Role op(?,?)

Rule 2 — Team, Goal, and Capability determine eligibilityback to ToC or SWRL ToC

If team has goal G requiring task T, and agent ∈ team and has capability C that allows T ⇒ agent eligible for T.

HI Team c(?) , has Goal op(?,?) , requires Task op(?,?) , has Member op(?,?) , has Capability op(?,?) , allows Task op(?,?) -> eligible For Task op(?,?)

Rule 3 — Eligibility implies assignmentback to ToC or SWRL ToC

If an agent is eligible for a task, then the agent can be assigned to that task.

eligible For Task op(?,?) -> assigned To Task op(?,?)

Rule 4 — Assignment → Performanceback to ToC or SWRL ToC

If an agent is assigned to a task and an execution realizes that task, then that execution is performed by the agent.

assigned To Task op(?,?) , realized By op(?,?) -> performed By op(?,?)
